# Medical AI Assistant — 현장 문제 해결형 의료영상 AI PoC

의료영상 AI는 검출 정확도만 높다고 바로 현장에서 쓰기 어렵습니다. 실제 사용에서는 결과를 이해할 수 있는 설명과, 설명을 뒷받침하는 근거 제시가 함께 필요합니다. 이 프로젝트는 이러한 간극을 줄이기 위해 YOLO 기반 병변 검출, VLM 기반 시각 해석, RAG 기반 근거 응답을 하나의 흐름으로 통합한 PoC입니다.

## 프로젝트 배경

의료 영상 분석 과정에서는 검출 결과가 좌표와 신뢰도 중심으로 제공되는 경우가 많아 비전문가가 결과를 해석하기 어렵고, 근거 문서를 별도로 찾아야 하는 부담이 발생합니다. 또한 근거 검색이 실패하면 설명 품질이 급격히 떨어져 결과 신뢰를 유지하기 어려워집니다. 본 프로젝트는 이 문제를 해결하기 위해 이미지 업로드 한 번으로 검출, 해석, 근거 확인까지 이어지는 구조를 목표로 설계되었습니다.

## 구현 내용

Kvasir-SEG와 DENTEX 데이터를 YOLO 세그멘테이션 학습 포맷으로 변환하고, 위장 및 치과 모델을 단일 서버에서 선택 실행할 수 있도록 통합 추론 경로를 구성했습니다. 여기에 VLM을 결합해 검출 결과를 자연어 설명으로 확장하고, RAG 파이프라인을 통해 관련 문헌 근거를 함께 제시하도록 구현했습니다. 내부 문서에서 근거를 찾지 못하는 경우에는 PubMed 웹 fallback을 통해 논문 제목, 저널, 연도, URL을 포함한 출처를 자동으로 보완하여 설명 공백을 줄이도록 설계했습니다.

## 성능 및 결과

Kvasir-SEG 기준으로 Mask mAP@50 0.942를 달성했으며, 검출 결과와 설명을 단일 흐름으로 제공해 결과 해석 단계를 단순화했습니다. 또한 근거 부족 상황에서도 PubMed 기반 출처를 자동 제시해 검증 가능한 응답을 유지하도록 개선했습니다. 이를 통해 단순 성능 데모가 아닌, 실제 도입을 고려한 설명 가능성과 재현성을 함께 확보했습니다.

## 기술 스택

Python, FastAPI, Ultralytics YOLOv8, LangChain LCEL, ChromaDB, Ollama(LLaVA), Docker, SQLite, pytest를 사용했습니다.

## 실행 방법

로컬 실행 시 의존성을 설치한 뒤 문서 인덱싱을 진행하고 서버를 실행하면 됩니다. Docker 환경에서도 동일한 흐름으로 실행 가능하도록 구성되어 있습니다.

```bash
pip install -r requirements.txt
python rag/ingest.py
python -m uvicorn api.app:app --reload --port 8000
```

필수 환경값은 `.env`에 설정하며, 로컬 LLM 모드에서는 Ollama 설정으로 OpenAI 의존 없이 동작할 수 있습니다.

## 주의사항

이 프로젝트는 의료진을 대체하는 진단 도구가 아니라 판독 보조를 위한 기술 검증용 PoC입니다. 실제 임상 환경 적용 시에는 기관 정책, 규제 요구사항, 데이터 보안 기준에 맞춘 별도 검증이 필요합니다.
